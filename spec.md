# player-list-scraper 仕様書

## 概要
オリコン業務向けの**プレイヤーリスト調査・正誤チェック自動化システム**。Streamlit GUI で4タブ構成。対象業界: クレジットカード(539件), 動画配信(36件), 中古車販売店(16件) 他。

## 機能一覧

### 1. プレイヤーの最新動向（v7.0 統合タブ）
3つのサブタブで既存リストの更新を一括管理:
- **変更点調査**: 撤退・統合・名称変更の自動検出（Gemini API）
- **新規参入検出**: 既存リストにない新規参入企業を自動検索
- **最新版リスト作成**: 調査結果を統合し最新版リストをエクスポート
- **業界定義**: 任意で業界の範囲・除外条件を指定しAI判定精度を向上

### 2. カテゴリチェック（テンプレート方式）
- 調査テンプレート管理: 作成・保存・再利用・インポート/エクスポート
- 組み込みテンプレート: 動画配信ジャンル、クレカブランド、8地方区分、47都道府県
- 判定基準（context）: LLMに具体的な判定ガイダンスを注入可能
- バッチプロンプト方式でコスト最適化

### 3. 店舗・教室調査
- AI調査（推奨）: LLMによるWeb検索ベースの調査
- 2段階ブランド発見 (v7.2): 0件時に企業名からブランド名特定、再調査を自動実行
- 都道府県別店舗・教室数: 概算数値で出力
- スクレイピング: サイト直接クローリング（3段階戦略）
- ハイブリッド: AI調査 + スクレイピング補完
- 安全性 (v7.2): robots.txt自動準拠、User-Agent識別、レート制限、監査ログ

### 4. 3段階チェック（ワークフロー）
- Phase 1: 実査前チェック（正誤全件 + 新規参入 + 属性全件）
- Phase 2: ランキング確定時チェック（正誤全件 + 属性変更分）
- Phase 3: 発表前チェック（CRITICALのみ再検証 + 最終確認）

### 5. 共通機能
- コスト表示: 日本円化 (USD_TO_JPY=150)
- LLMキャッシュ (TTL付き)
- Excel読み書き（複数シート対応、load_multiple）
- アラートレベル: 緊急(赤)/警告(黄)/情報(緑)/正常

## 技術スタック

| カテゴリ | 技術 |
|---------|------|
| GUI | Streamlit (Streamlit Cloud デプロイ済み) |
| LLM | Gemini API (`gemini-2.5-pro`, `google-genai`) |
| データ | openpyxl (Excel), 複数シート対応 |
| テスト | pytest (522件) |
| 非同期 | asyncio |
| セキュリティ | html.escape, SSRF防止, robots.txt準拠 |
| 認証 | パスワード認証 (hmac.compare_digest) |

## 非機能要件
- テスト: 522件全パス
- セキュリティ: XSS防止、SSRF防止、プロンプトインジェクション防御、パストラバーサル防止
- 合法性: robots.txt自動準拠、User-Agent識別、レート制限、監査ログ
- 認証: Streamlit Cloud でパスワード認証（試行回数制限5回）
- ロギング: 日次ローテーション、30日保持

## 用語定義

| 用語 | 説明 |
|------|------|
| プレイヤー | 調査対象の企業・サービス |
| 正誤チェック | 既存リストの撤退・統合・名称変更を検出 |
| 新規参入検出 | 既存リストにない新規参入企業を検索 |
| 3段階チェック | 実査前/確定時/発表前の段階的検証ワークフロー |
| テンプレート | 属性調査の再利用可能な定義セット |
| context | LLMに注入する判定基準テキスト |
| definition | 業界の範囲・除外条件を定義するテキスト |
